1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

git checkout "name of file"

2.Как вернуть файл из индекса в рабочий каталог?

git restore --staged "name of file"

3.Как исправить, отменить  ошибочный commit ?

git commit --amend -m "Your comment"
git reset -HEAD~1 - отменить commit на опреденый уровень

4.Как исправить коммит отправленный на сервер GitHub?

git revert HEAD~

5.Как создать новую ветку и переключится в нее?

git checkout -b + name of branch


5.В чем отличение между fast-forward merge  and non fast-forfard merge?

Fast- forward - линейная история, дополняет в перемещаюмую ветку дополнительные коммиты из другой ветки.
NON Fast-forward - на обеих ветках была произведенна работа (commit).Git создаёт через редактор новый commit.


6.Когда может возникнуть конфликт слияния веток, как  его разрешить?

*Когда изменения были сделанны в обеих ветках (в одном и том же файле).
* Открыть редактор и открыть файл с Конфликтом. Удалить указатели. Удалить в текстовом редакторе файлы которые уже есть на нашей ветке. Сохранить наш исправленный файл. 
Если файлы нам нужны и ветки соединять, то git commit -a "",
если не нужны, то git merge --abort


7.Можно ли удалить ветку? Как? 
git branch -d + name of branch.
если есть данные но нам они не нужны - git branch -D + name of branch.
